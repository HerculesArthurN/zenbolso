# Plan 01-01: Responsive Layout System

## Objective
Replace the single-mode mobile container (`max-w-[430px]` centered shell in `App.tsx`) with a responsive layout that uses the full viewport on desktop (≥768px) while preserving the mobile shell experience on small screens.

## Context / Why

Currently `App.tsx` wraps everything in a `max-w-[430px]` div that makes the app look like a phone floating on a grey background regardless of screen size. On desktop this wastes 70%+ of available space and makes interactions awkward with mouse.

The desktop layout should feel like a real web app: full width, no forced mobile container.

## Tasks

### Task 1 — Refactor App.tsx container strategy

<read_first>
- `src/App.tsx` — current provider tree and container structure
- `src/components/layout/AppLayout.tsx` — layout wrapper used by routed pages
- `tailwind.config.js` — custom breakpoints and tokens
</read_first>

<action>
Replace the current single `max-w-[430px]` container in `App.tsx` with a responsive strategy:

```tsx
// BEFORE (in App.tsx) — single mobile shell:
<div className="fixed inset-0 min-h-screen bg-zinc-950 flex items-center justify-center overflow-hidden font-sans selection:bg-teal-500/30">
  {/* ambient blobs */}
  <div className="relative w-full max-w-[430px] h-full sm:h-[92vh] sm:max-h-[900px] bg-zinc-900 sm:rounded-[40px] shadow-[0_32px_64px_-15px_rgba(0,0,0,0.6)] border-0 sm:border-[8px] border-zinc-900 overflow-hidden flex flex-col z-10 transition-all duration-500">
    <div className="flex-1 overflow-y-auto no-scrollbar scroll-smooth relative">
      <AppRoutes />
    </div>
  </div>
</div>

// AFTER — responsive: mobile shell on sm, full layout on md+
<div className="min-h-screen bg-zinc-950 font-sans selection:bg-teal-500/30">
  {/* Mobile: centered phone shell (unchanged look) */}
  <div className="md:hidden flex items-center justify-center min-h-screen overflow-hidden relative">
    {/* Ambient blobs — only on mobile shell */}
    <div className="absolute top-0 left-0 w-full h-full overflow-hidden pointer-events-none opacity-40">
      <div className="absolute top-[-10%] left-[-10%] w-[50%] h-[50%] bg-teal-600/20 rounded-full blur-[120px] animate-blob" />
      <div className="absolute bottom-[-10%] right-[-10%] w-[50%] h-[50%] bg-indigo-600/20 rounded-full blur-[120px] animate-blob animation-delay-2000" />
    </div>
    <div className="relative w-full max-w-[430px] h-screen bg-zinc-900 overflow-hidden flex flex-col z-10">
      <div className="flex-1 overflow-y-auto no-scrollbar scroll-smooth relative">
        <AppRoutes />
      </div>
    </div>
  </div>

  {/* Desktop: full viewport layout (AppLayout handles sidebar) */}
  <div className="hidden md:block">
    <AppRoutes />
  </div>
</div>
```

Keep `ReloadPrompt`, `InstallPrompt`, `OnboardingWizard`, `AppLockScreen` outside both blocks so they render in both modes.
</action>

<acceptance_criteria>
- `App.tsx` contains `md:hidden` on the mobile shell div
- `App.tsx` contains `hidden md:block` on the desktop block
- `App.tsx` does NOT have a single outer `max-w-[430px]` that wraps both mobile and desktop
- At viewport width ≥768px: the app renders without the phone-shell border-radius
- `AppLockScreen` and `OnboardingWizard` are outside the responsive split (render in both)
</acceptance_criteria>

### Task 2 — Update AppLayout.tsx for desktop full-width

<read_first>
- `src/components/layout/AppLayout.tsx` — current layout (21 lines, mobile-only)
- `src/components/layout/BottomNav.tsx` — bottom nav to be hidden on desktop
</read_first>

<action>
Update `AppLayout.tsx` to support desktop layout. On desktop (md+), the main content area should not have `pb-24` (no bottom nav clearance) and should take full width:

```tsx
// AFTER AppLayout.tsx
import React from 'react';
import { BottomNav } from './BottomNav';
import { DesktopSidebar } from './DesktopSidebar'; // Created in Plan 01-02
import { Outlet } from 'react-router-dom';

export const AppLayout: React.FC = () => {
  return (
    <div className="min-h-screen bg-zinc-900 text-zinc-100 transition-colors duration-300">
      {/* Desktop: sidebar + main content side by side */}
      <div className="hidden md:flex h-screen">
        <DesktopSidebar />
        <main className="flex-1 overflow-y-auto">
          <div className="max-w-5xl mx-auto px-6 py-8 animate-in fade-in duration-300">
            <Outlet />
          </div>
        </main>
      </div>

      {/* Mobile: original bottom-nav layout */}
      <div className="md:hidden">
        <main className="pb-24 min-h-[100vh]">
          <div className="w-full px-2 py-6 animate-in fade-in duration-300">
            <Outlet />
          </div>
        </main>
        <BottomNav />
      </div>
    </div>
  );
};
```

Note: `DesktopSidebar` is created in Plan 01-02. If this plan runs first, use a placeholder import that renders `null` temporarily.
</action>

<acceptance_criteria>
- `AppLayout.tsx` contains `hidden md:flex` for desktop layout block
- `AppLayout.tsx` contains `md:hidden` for mobile layout block
- `BottomNav` is only rendered inside the `md:hidden` block
- `DesktopSidebar` is imported and rendered inside the desktop block
- File stays under 200 lines
</acceptance_criteria>
